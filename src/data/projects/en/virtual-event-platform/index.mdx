## Project Overview

A mid-size company was running their analytics on an aging on-premise SQL Server warehouse — 300+ stored procedures, no version control, no documentation, minimal testing. Queries that should have run in seconds took minutes. The data team spent 60% of their time on incident response rather than new work.

I scoped, planned, and led the migration to BigQuery with a modern Python transformation layer over 12 weeks.

## Technical Implementation

Phase 1: Fivetran connectors for all source systems, landing raw data into BigQuery staging datasets. Phase 2: Python transformation modeling — staging, intermediate, and mart layers built iteratively, validated against the legacy stored procedure outputs row by row. Phase 3: Airflow DAGs replacing SSIS packages for orchestration, with alerting on failed runs.

The most technically challenging component was the incremental logic. Several legacy procedures had implicit state that was never documented. I reverse-engineered the behavior, documented the intent, and rebuilt it as transparent, tested Python incremental models with clear audit trails.

## Outcome

Query performance improved 10-15x on average analytical workloads. Team incident response time dropped from hours to minutes for most data issues. The data team shipped more new projects in Q1 post-migration than in the previous two quarters combined.
